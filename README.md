# 統計検定道場 (Statistics Exam Preparation App)

統計検定 準1級・2級・1級の学習を支援する Web アプリケーションです。
教科書（KaTeX 数式対応）、問題演習、模擬試験、学習進捗管理の機能を備え、Supabase によるユーザー認証とデータ永続化、Claude Haiku による AI 採点・チャットに対応しています。

**本番 URL**: https://statistics-exam-preparation-app.vercel.app/

---

## 機能一覧

| 機能 | 概要 |
|------|------|
| 教科書 | 23 トピック × 10 例題。KaTeX 数式レンダリング、解答の折りたたみ表示 |
| 問題演習 | 276 問（準1級）の選択式・記述式。トピック別・難易度別フィルタ、各選択肢の解説付き |
| 難易度タグ | 全問に基礎・標準・発展の 3 段階を付与。段階的な学習が可能 |
| 模擬試験 | タイマー付き試験（115 問）。自動採点、クイックナビ、結果レビュー |
| ブックマーク | 問題をブックマークし、ブックマーク復習モードで集中的に解き直し |
| 弱点分析 | 正答率 60%未満のカテゴリを自動検出・ハイライト。未着手カテゴリも一覧表示 |
| 記述式 AI 評価 | AI ON → Claude Haiku が 0〜100 点で採点 + フィードバック。OFF → キーワード採点 |
| AI チャット | AI ON → Claude による自由質問応答。OFF → 教科書コンテンツ検索 + 72 キーワード辞書（オフライン可） |
| 学習進捗 | トピック別正答率、回答履歴、連続学習日数（ストリーク） |
| ユーザー管理 | Supabase Auth（メール+パスワード）/ ゲストモード |
| プロフィール | 表示名編集、学習進捗リセット |

## AI 機能

### アーキテクチャ

```
ブラウザ (React SPA)
  ├─ [AI] チェックボックス OFF → ローカル処理（無料・オフライン可）
  │   ├─ 記述式: キーワード部分一致で自動採点
  │   └─ チャット: 教科書コンテンツ検索 → 72 キーワード辞書マッチ
  │
  └─ [AI] チェックボックス ON → Vercel API Route (/api/ai) → Claude Haiku
      ├─ 記述式: 0〜100 点のスコア + フィードバック + 良い点/不足点
      └─ チャット: 統計学の自由質問に対する文脈ある回答
          （API 失敗時は自動でローカルにフォールバック）
```

ヘッダー右上の **[AI] チェックボックス**で切り替え。設定は localStorage に保存されます。

### API Route (`POST /api/ai`)

| 項目 | 内容 |
|------|------|
| ランタイム | Vercel Serverless Function (Node.js) |
| リクエスト種別 | `type: "evaluate"`（記述式評価）/ `type: "chat"`（チャット応答） |
| 認証 | Supabase JWT を Authorization ヘッダーで送信 |
| モデル | `claude-haiku-4-5-20251001` |
| APIキー | `ANTHROPIC_API_KEY`（サーバー側のみ、ブラウザに露出しない） |

### コスト管理・利用制限

| 項目 | 値 |
|------|-----|
| 1 リクエストあたりのコスト | ~0.5 円（入力 ~1500 tok + 出力 ~400 tok） |
| デフォルト月間上限 | **200 回/ユーザー（≈ 100 円/月）** |
| ユーザー別上限の変更 | `profiles.ai_monthly_limit` カラムで個別設定可能 |
| 上限到達時 | 429 エラー + メッセージ表示、翌月 1 日にリセット |
| 短期レート制限 | 20 回/分/IP（DDoS 防止） |
| 未ログイン・ゲスト | AI 機能利用不可 |

利用量は `ai_usage` テーブルで `(user_id, month)` 単位で管理。記述式評価とチャットは**同一の月間上限を共有**します。

#### 特定ユーザーの上限変更

Supabase SQL エディタで実行：

```sql
-- 上限を 500 回/月に引き上げ
UPDATE profiles SET ai_monthly_limit = 500
WHERE email = 'user@example.com';

-- 無制限にする（実質無制限の大きな値を設定）
UPDATE profiles SET ai_monthly_limit = 99999
WHERE email = 'user@example.com';

-- デフォルト（200 回）に戻す
UPDATE profiles SET ai_monthly_limit = NULL
WHERE email = 'user@example.com';
```

`ai_monthly_limit` が `NULL` のユーザーにはデフォルト 200 回が適用されます。

### チャットボットのローカル応答（AI OFF 時）

AI 無効時の応答は以下の優先順位で生成されます：

1. **教科書コンテンツ検索** — 全トピックのセクションタイトル・本文をスコアリングし、最も関連するセクションの内容を抜粋して返す（KaTeX 数式付き、「教科書より」バッジ表示）
2. **キーワード辞書** — `src/data/chat-responses.js` の 72 キーワードで部分一致
3. **デフォルトメッセージ** — キーワード例を提示

| カテゴリ | キーワード例 |
|---------|------------|
| 基礎統計 | 分散、標準偏差、平均、中央値、相関 |
| 確率分布 | 正規分布、ポアソン分布、t分布、カイ二乗、F分布 |
| 統計的推測 | 信頼区間、p値、検定、t検定、z検定 |
| 準1級（推測） | 最尤法、フィッシャー情報量、EM、ベイズ、ブートストラップ |
| 準1級（過程） | マルコフ、ブラウン運動、伊藤、ポアソン過程、積率母関数 |
| 準1級（回帰） | 回帰、重回帰、ロジスティック、ANOVA、グラフィカルモデル |
| 準1級（多変量） | 主成分分析、因子分析、判別分析、クラスター、多変量正規 |
| 準1級（時系列） | 時系列、定常性、ARIMA、生存分析、コックス |
| 準1級（調査） | 標本調査、層化抽出、分割表、不完全データ、多重代入 |
| 数学的基礎 | 中心極限定理、大数の法則、デルタ法、十分統計量 |

クイック質問は選択中の級に応じて変化（2級6個、準1級8個、1級6個）。

### テーブル一覧

| テーブル | 用途 | 主なカラム |
|---------|------|-----------|

## セキュリティ

| 項目 | 実装 |
|------|------|
| 認証 | Supabase Auth（メール + パスワード）。未設定時は localStorage フォールバック |
| パスワード要件 | 8 文字以上、大文字・小文字・数字各 1 つ以上 |
| RLS | 全テーブルに Row Level Security。ユーザーは自分のデータのみアクセス可 |
| API キー保護 | `ANTHROPIC_API_KEY`, `SUPABASE_SERVICE_ROLE_KEY` はサーバー側のみ |
| AI 利用量制限 | `ai_usage` テーブルでサーバー側管理。ユーザー別上限対応 |
| CSP | `default-src 'self'`、Supabase・GA・自身の API のみ外部接続許可 |
| HSTS | `max-age=63072000; includeSubDomains; preload` |
| XSS | React JSX エスケープ。`dangerouslySetInnerHTML` 不使用 |

## 運用

### 本番安全ルール

- **DB スキーマ変更**: `CREATE TABLE IF NOT EXISTS` / `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` のみ使用。`DROP TABLE` 禁止
- **question ID**: `results` に既存 `question_id` が記録済み。ID の変更・削除は不可（追加は可）
- **export 名**: `QUESTIONS`, `EXAM_QUESTIONS`, `ALL_QUESTIONS`, `TOPICS` を変更すると全コンポーネントが壊れる
- **Service Role Key**: サーバー側のみ。クライアントに露出させない
- **main ブランチ**: push 即デプロイ。既存ユーザーに直接影響する

### 教科書コンテンツの記法

| 記法 | 用途 |
|------|------|
| `$...$` / `$$...$$` | インライン / ディスプレイ数式 |
| `**...**` | 太字見出し |
| `【例題】` | 問題マーカー（青ボックス） |
| `【解答】...【/解答】` | 折りたたみ解答 |
